# Guía de contribución

En primer lugar: ¡gracias por dedicar tu tiempo a contribuir!

Se valora todo tipo de contribuciones. Y si te gusta el proyecto, pero no cuentas con tiempo para contribuir, no hay problema. Puedes apoyar el proyecto de otras formas más sencillas:
> - Marca el proyecto como favorito.
> - Comparte el proyecto en tus redes sociales.
> - Menciona este proyecto en el archivo README de tu proyecto.
> - Habla del proyecto con tus conocidos/gente local.

## Código de conducta

Al participar, se espera que respetes este código. Por favor, informa sobre cualquier comportamiento inaceptable a <reemplazar@gmail.com>

- No trabajar directamente sobre main
- Crear una rama por funcionalidad o incremento
- Realizar Pull Request para integrar cambios
- Los PRs se revisan antes de hacer merge

## Tengo una pregunta
> Si deseas hacer una pregunta, asumimos que has leído la documentación disponible.

Antes de preguntar, es recomendable buscar respuestas en internet primero.

Si aún tienes alguna pregunta o necesitas alguna aclaración, te recomendamos que sigas los siguientes pasos:

- Abre una incidencia.
- Proporciona toda la información posible sobre el problema.
- Indica las versiones del proyecto.

Nos ocuparemos de la incidencia lo antes posible.

## Informar sobre errores
Un buen error no debería obligar a otros a contactarte para obtener más información. Por esto, te pedimos que investigues, recopiles información y describas el problema con detalle en tu informe. Por favor, siga estos pasos para ayudarnos a solucionar cualquier error lo antes posible.

- Asegúrese de estar utilizando la última versión.
- Determine si el error es un fallo de sistema y no un problema de su parte, por ejemplo, el uso de componentes o versiones de entorno incompatibles (asegúrese de haber leído la documentación).
- Para ver si otros han experimentado el mismo problema, compruebe si ya existe un informe del error.
- ¿Puede reproducir el problema de forma fiable?

> Nunca debe informar sobre problemas de seguridad, vulnerabilidades o errores que incluyan información confidencial en el sistema de seguimiento de errores ni en ningún lugar público. Estos deben enviarse por correo a <reemplazar@gmail.com>.
